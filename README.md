## وصف المنظومة

DCMI ENTERPRISE هي منظومة متكاملة لإدارة شئون الأفراد العسكريين، تعمل بالكامل على المتصفح دون الحاجة لخادم خلفي. تتيح المنظومة إدارة قواعد بيانات الأفراد، متابعة الحضور والغياب، إدارة الإجازات، الترقيات، إصدار النماذج الرسمية، وتوليد التقارير الاحترافية.

صُممت المنظومة لتكون آمنة، سريعة، وسهلة الاستخدام في البيئات العسكرية التي تتطلب الدقة والموثوقية.

---

## الميزات الرئيسية

- **ادارة القوة البشرية** — سجلات شاملة للأفراد تشمل الرتبة، الوحدة، الحالة الوظيفية، المستندات، والتاريخ الوظيفي
- **التمام اليومي** — تسجيل الحضور والغياب مع تتبع حالة كل فرد في الوقت الفعلي
- **الاجازات والاذونات** — نظام متكامل لطلبات الاجازة مع الموافقة والمتابعة
- **ادارة الترقيات** — تتبع الترقيات وتاريخ الرتب مع قواعد الاستحقاق
- **النماذج الرسمية** — توليد نماذج معتمدة بصيغة PDF جاهزة للطباعة مع ختم وتوقيع
- **التقارير والاحصائيات** — تقارير شاملة مع رسوم بيانية وتصدير Excel/PDF
- **لوحة التحكم** — ملخص فوري لمؤشرات الوحدة والجاهزية القتالية
- **ادارة المستخدمين** — صلاحيات متعددة المستويات (مشرف، مدير، موظف)
- **النسخ الاحتياطي** — تصدير واستيراد كامل لبيانات المنظومة بصيغة JSON
- **وضع الصيانة** — تجميد الوصول لجميع المستخدمين ما عدا المشرف

|---------|---------------|

## خطوات التثبيت والتشغيل

```bash
# 1. استنساخ المستودع
git clone <repository-url>
cd military_hr_system

# 2. تثبيت الاعتماديات
npm install

# 3. (اختياري) ضبط مفتاح Gemini API في ملف البيئة
echo "GEMINI_API_KEY=your_key_here" > .env.local

# 4. تشغيل خادم التطوير
npm run dev
# المنظومة ستعمل على: http://localhost:3000

# 5. بناء نسخة الانتاج
npm run build
```

**بيانات الدخول الافتراضية:**
- اسم المستخدم: `admin`
- كلمة السر: `123`

## ملاحظات الامان

- **تشفير كلمات المرور** — تُخزن كلمات المرور باستخدام `bcryptjs` ولا تظهر في نص واضح ابدا
- **حذف كلمة المرور من الجلسة** — عند تخزين بيانات المستخدم الحالي يُحذف حقل `password` تلقائيا
- **Rate Limiting** — بعد 5 محاولات دخول فاشلة، يُقفل النظام لمدة 5 دقائق مع عداد تنازلي مرئي
- **مسح الجلسة عند الخروج** — `clearSession()` تمسح بيانات الجلسة من الذاكرة والتخزين المحلي
- **بيانات تجريبية محمية** — البيانات التجريبية (Mock Data) تعمل فقط عند تفعيل `devMode` في الاعدادات
- **النسخ الاحتياطية آمنة** — ملفات التصدير لا تحتوي على كلمات المرور
- **Error Boundaries** — كل تبويب محاط بـ Error Boundary لمنع انهيار التطبيق بالكامل عند حدوث خطأ
- **وضع الصيانة** — يمكن تجميد الوصول فوريا من اعدادات الامان
