# توثيق الانتقال إلى قاعدة البيانات

## التغييرات المنفذة

### 1. قاعدة البيانات (SQLite)
- تم إنشاء قاعدة بيانات SQLite باستخدام `better-sqlite3`
- ملف قاعدة البيانات: `/data/military.db`
- جداول قاعدة البيانات:
  - `users`: بيانات المستخدمين والصلاحيات
  - `personnel`: سجلات الأفراد العسكريين
  - `leaves`: طلبات الإجازات والأذونات
  - `attendance`: الحضور والغياب اليومي
  - `promotions`: الترقيات والرتب
  - `audit_logs`: سجل التدقيق الأمني
  - `settings`: إعدادات النظام

### 2. الخادم (Express.js)
- تم إنشاء خادم Express على المنفذ 3001
- نقاط API المتاحة:
  - `POST /api/auth/login`: تسجيل الدخول
  - `GET /api/auth/me`: الحصول على بيانات المستخدم الحالي
  - `GET/POST/PUT/DELETE /api/personnel`: إدارة الأفراد
  - `GET/POST/PUT /api/leaves`: إدارة الإجازات
  - `GET/POST /api/attendance`: إدارة الحضور
  - `GET/PUT /api/settings`: إدارة الإعدادات
  - `GET /api/stats/dashboard`: إحصائيات لوحة التحكم

### 3. تقسيم الكود (Code Splitting)
تم تكوين Vite لتقسيم الكود تلقائياً:
- `vendor`: مكتبات React الأساسية
- `charts`: مكتبة Recharts للرسوم البيانية
- `animations`: مكتبة Framer Motion للحركة
- `utils`: أدوات المساعدة (XLSX, QRCode, etc.)
- `icons`: أيقونات Lucide

### 4. عميل API
- تم إنشاء `src/api/apiClient.ts` للاتصال بالخادم
- يدعم جميع عمليات CRUD
- يدير المصادقة تلقائياً عبر sessionStorage

## التشغيل

### تطوير محلي (خادم + واجهة):
```bash
npm run dev
```

هذا سيشغل:
- خادم Express على http://localhost:3001
- واجهة Vite على http://localhost:5173

### بناء للإنتاج:
```bash
npm run build
```
